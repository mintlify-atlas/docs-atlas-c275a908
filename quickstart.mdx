---
title: "Quick Start"
description: "Get VITACORE running in minutes — from cloning the repository to your first login."
---

<Note>
  You need a [Supabase](https://supabase.com) project with PostgreSQL and Auth enabled before running VITACORE. The application uses Supabase for all data persistence and user authentication.
</Note>

<Steps>
  <Step title="Clone the repository">
    Download the source code to your local machine:

    ```bash
    git clone https://github.com/msgiraldo01-wq/VITACORE.git
    cd VITACORE
    ```
  </Step>

  <Step title="Create a virtual environment and install dependencies">
    VITACORE requires Python 3. Create a virtual environment and install the required packages:

    <CodeGroup>
    ```bash macOS / Linux
    python3 -m venv venv
    source venv/bin/activate
    pip install -r requirements.txt
    ```

    ```bash Windows
    python -m venv venv
    venv\Scripts\activate
    pip install -r requirements.txt
    ```
    </CodeGroup>

    The `requirements.txt` installs:

    | Package | Purpose |
    |---|---|
    | `flask` | Web framework and routing |
    | `supabase` | Supabase Python client |
    | `python-dotenv` | Loads environment variables from `.env` |
  </Step>

  <Step title="Create your .env file">
    Copy the template below into a file named `.env` at the project root and fill in your Supabase credentials:

    ```bash .env
    FLASK_SECRET_KEY=your-secret-key-here
    SUPABASE_URL=https://your-project-id.supabase.co
    SUPABASE_ANON_KEY=your-anon-key
    SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
    SUPABASE_TABLE_HC_PROFESIONALES=hc_profesionales
    ```

    | Variable | Where to find it |
    |---|---|
    | `FLASK_SECRET_KEY` | Any long random string — used to sign session cookies |
    | `SUPABASE_URL` | Supabase dashboard → Project Settings → API → Project URL |
    | `SUPABASE_ANON_KEY` | Supabase dashboard → Project Settings → API → `anon` public key |
    | `SUPABASE_SERVICE_ROLE_KEY` | Supabase dashboard → Project Settings → API → `service_role` key |
    | `SUPABASE_TABLE_HC_PROFESIONALES` | Name of the professionals table (default: `hc_profesionales`) |

    <Warning>
      Never commit your `.env` file to version control. Add it to `.gitignore` to prevent accidentally exposing your Supabase credentials.
    </Warning>
  </Step>

  <Step title="Start the development server">
    ```bash
    python app.py
    ```

    You should see Flask start on port 5000:

    ```
     * Running on http://127.0.0.1:5000
     * Debug mode: on
    ```
  </Step>

  <Step title="Open the application in your browser">
    Navigate to [http://localhost:5000](http://localhost:5000). VITACORE checks for an active session and redirects unauthenticated users to the login page at `/auth/login`.
  </Step>

  <Step title="Log in">
    Enter your **username** and **password**. VITACORE performs a two-phase login:

    1. Looks up the username in the `hc_profesionales` table to retrieve the associated email address.
    2. Calls `supabase.auth.sign_in_with_password()` with the email and password.

    On success, your session is populated with your profile and role information and you are redirected to the dashboard.

    <Tip>
      The first user must be created directly in Supabase Auth and the `hc_profesionales` table, or through the user management interface at `/hc/configuracion/usuarios/` if an admin account already exists.
    </Tip>
  </Step>
</Steps>
