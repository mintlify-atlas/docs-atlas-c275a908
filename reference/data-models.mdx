This page documents the normalized data shapes returned by VITACORE repositories. Each repository applies a `_normalize_row()` function before returning data to route handlers, so the shapes below are what you can expect in Jinja2 templates and any downstream logic.

## User Profile (`profiles`)

The profile shape is built from the `profiles` table joined with `roles`. You access it via `session["user"]` in templates and route handlers.

| Field | Type | Description |
|---|---|---|
| `id` | string (UUID) | Matches `auth.users.id` in Supabase Auth |
| `username` | string | Login username (lowercase) |
| `full_name` | string | Display name |
| `email` | string | Email address |
| `role_id` | integer | FK to the `roles` table |
| `role` | string | Role machine code (from `roles.code`) |
| `role_name` | string | Role display name (from `roles.name`) |

Additionally, `session["supabase_access_token"]` and `session["supabase_refresh_token"]` hold the raw Supabase Auth JWT tokens.

## Local JSON data files

The `data/` directory contains flat JSON files used by the `*_local.py` repository variants (file-based fallback, no Supabase required). Each file stores an array of objects matching the structure of its corresponding Supabase table.

| File | Corresponds to | Default content |
|---|---|---|
| `data/hc_sedes.json` | `hc_sedes` table | `[]` |
| `data/hc_tipos_documento.json` | `hc_tipos_documento` table | `[]` |
| `data/hc_especialidades.json` | `hc_especialidades` table | `[]` |
| `data/hc_consultorios.json` | `hc_consultorios` table | `[]` |
| `data/hc_profesionales.json` | `hc_profesionales` table | `[]` |

The local repos auto-create missing files on first read. An example `hc_sedes.json` entry produced by `hc_sedes_repo_local.crear()` looks like this:

```json
[
  {
    "id": 1,
    "codigo": "SEDE01",
    "nombre": "Sede Principal",
    "ciudad": "Bogotá",
    "direccion": "Calle 123 # 45-67",
    "telefono": "6015551234",
    "estado": "ACTIVA",
    "created_at": "2025-01-15T10:30:00",
    "updated_at": null
  }
]
```
