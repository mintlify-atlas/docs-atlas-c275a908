---
title: "Configuration Routes"
description: "Administrative configuration routes for facilities, catalogues, users, roles, and route sync."
---

VITACORE's configuration module lives under the `/hc/configuracion` prefix (blueprint `hc_configuracion`) and two additional blueprints: `configuracion_usuarios` (`/hc/configuracion/usuarios`) and `configuracion_roles` (`/hc/configuracion/roles-permisos`). An admin utility route is registered on `bp_admin` under `/admin`.

All configuration routes use the admin Supabase client and require an authenticated session. Role-permission routes additionally require the `@rol_required(usar_matriz=True)` decorator.

---

## Sedes (Facilities)

| Method | Path | Description |
|---|---|---|
| `GET` | `/hc/configuracion/sedes` | List all facilities |
| `GET / POST` | `/hc/configuracion/sedes/nueva` | Create a new facility |
| `GET / POST` | `/hc/configuracion/sedes/editar/<sede_id>` | Edit an existing facility |
| `POST` | `/hc/configuracion/sedes/toggle/<sede_id>` | Toggle facility status (`ACTIVA` / `INACTIVA`) |

**Create / edit form fields:** `codigo` (required, unique), `nombre` (required), `ciudad`, `direccion`, `telefono`, `estado`

## Usuarios (Users)

Managed by the `configuracion_usuarios` blueprint at `/hc/configuracion/usuarios`.

| Method | Path | Description |
|---|---|---|
| `GET` | `/hc/configuracion/usuarios/` | List all users and render inline create form |
| `POST` | `/hc/configuracion/usuarios/` | Create a new user with Supabase Auth account |
| `GET / POST` | `/hc/configuracion/usuarios/<user_id>/editar` | Edit user profile and role |
| `POST` | `/hc/configuracion/usuarios/<user_id>/estado` | Toggle user `is_active` status |

**Create form fields:** `username`, `full_name`, `email`, `password` (min 6 chars), `role_id`

**Edit form fields:** `username`, `full_name`, `email`, `role_id`, `is_active`

<Info>
  User creation calls `crear_usuario_con_perfil()`, which first creates the account in Supabase Auth (`auth.admin.create_user`) and then inserts a matching row in the `profiles` table using the returned user UUID.
</Info>

## Roles y Permisos (Roles & Permissions)

Managed by the `configuracion_roles` blueprint at `/hc/configuracion/roles-permisos`.

| Method | Path | Description |
|---|---|---|
| `GET` | `/hc/configuracion/roles-permisos/` | View permission matrix for the selected role |
| `POST` | `/hc/configuracion/roles-permisos/` | Save updated permissions for the selected role |
| `POST` | `/hc/configuracion/roles-permisos/crear` | Create a new role |

**Permission matrix fields (per module):** `modulo_ids[]`, `view_{modulo_id}`, `create_{modulo_id}`, `edit_{modulo_id}`, `delete_{modulo_id}`

The save operation deletes the existing `roles_modulos` rows for the role and inserts the new permission set in a single transaction.

**Create role fields:** `code` (lowercase, required), `name` (required)

## Admin: Route Sync

| Method | Path | Description |
|---|---|---|
| `GET` | `/admin/sync_rutas` | Sync all Flask endpoints to the Supabase `rutas` table |

See [Supabase Integration — Route sync](/integrations/supabase#route-sync) for the full implementation details.

<Warning>
  The `/admin/sync_rutas` endpoint uses the service role key and has no authentication guard in the default implementation. Protect this endpoint with network-level access controls or add a session check before deploying to production.
</Warning>
